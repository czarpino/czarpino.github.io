How to compile PHP

... for an Apache web server.

This is a how-to for compiling PHP as a module of Apache so that it may be used as the PHP interpreter of an Apache web server. Before proceeding, be sure to have a properly compiled Apache web server first. Better yet, compile your own Apache web server.

By the way, this is for OSX.

Organizing

Firstly, prepare a build directory to house the PHP source codes and an installation directory where PHP will be installed. 

mkdir -p ~/build/php5.5    # for sources
mkdir -p ~/tools/php5.5    # for compiled files

Get Source

Next, download the PHP source files. At the time of writing it is available from http://php.net/downloads.php. I'd recommend the gzipped (.gz) version as it worked for me.

Now, assuming the downloaded file went to ~/Downloads, extract the files and move them to the build directory. 

cd ~/Downloads
tar -xzvf php-5.5.0.tar.gz
mv php-5.5.0 ~/build/php5.5
cd ~/build/php5.5

Configuration

From the build directory, the next step is now to configure compilation and installation settings. 

./configure --prefix=$HOME/tools/php/php-5.5 \ 
--with-apxs2=$HOME/tools/apache/apache-2.4/bin/apxs \
--enable-cli --enable-intl --with-mcrypt --with-pdo-mysql \
--with-mysql --with-curl --with-curl --with-gd --with-jpeg-dir \
--enable-bcmath --enable-calendar --with-openssl --with-xsl

The code above is just my own set of irresponsible configurations. I only use this for development; not production.

Building

After configuring, the sources can now be compiled. Run make to compile PHP and generate the files to be installed.

make clean
make

This step usually takes some time...

Installation

Finally, PHP has been configured and built. All that's left is now to install PHP.

make install
